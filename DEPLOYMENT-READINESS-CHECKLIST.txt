PRE-DEPLOYMENT SETUP
═════════════════════════════════════════════════════════════════════════════════

1. Initialize Production Environment:
   bash scripts/setup-production-env.sh
   
   This will:
   - Generate cryptographic secrets (JWT_SECRET, ENCRYPTION_KEY, CSRF_SECRET)
   - Create necessary directories
   - Validate environment configuration
   - Setup monitoring recommendations

2. Configure Environment Variables:
   - Add Firebase credentials to .env.production
   - Set ALLOWED_ORIGIN to your production domain
   - Configure any external service tokens
   - Set up secrets in Vercel Environment Variables UI

3. Run Security Validation:
   npm run audit:security
   bash scripts/security-validation.sh

4. Make Scripts Executable (Linux/Mac):
   bash scripts/make-executable.sh

ROLLBACK PROCEDURE
═════════════════════════════════════════════════════════════════════════════════

If issues are detected:

1. Immediate Rollback:
   vercel rollback  (if using Vercel)

2. Backup Recovery:
   - Backups are stored in .backups/ directory
   - Restore from backup: cp -r .backups/[timestamp]/* .

3. Database Rollback:
   - Use Firebase automated backups
   - Restore to point-in-time if needed

4. Code Rollback:
   git revert [commit-hash]
   git push
   npm run deploy:production
