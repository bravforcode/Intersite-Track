# รายงานรายละเอียดการพัฒนาระบบแจ้งเตือน LINE
## Intersite Track (TaskAm)

**ขอบเขตเอกสาร:** สรุปเฉพาะงานที่เกี่ยวข้องกับระบบแจ้งเตือน LINE, วันหยุด, เวรวันเสาร์ และงานสนับสนุนที่ทำให้ระบบแจ้งเตือนใช้งานได้จริง  
**รูปแบบเอกสาร:** ปรับให้อยู่ในรูปแบบที่นำไปเขียนรายงานสหกิจต่อได้ โดยคงหัวข้อ `งาน / ผลลัพธ์ / หลักฐาน / ภาพประกอบ` ตามแนวตัวอย่างรายงาน  
**หมายเหตุ:** ตัดรายละเอียดเชิงออกแบบ UI ออกตามขอบเขตที่ต้องการ แต่ยังเก็บผลลัพธ์ของหน้าจอที่เกี่ยวข้องไว้ในส่วนหลักฐานและภาพประกอบ

---

## บทที่ 1 บทนำ

### 1.1 ภาพรวมของงานที่รับผิดชอบ
ในช่วงพัฒนาโปรเจกต์ Intersite Track ได้มีการเพิ่มระบบแจ้งเตือนผ่าน LINE เพื่อช่วยให้ผู้ใช้งานรับรู้ข้อมูลสำคัญของงานได้ทันทีโดยไม่ต้องเปิดระบบตลอดเวลา งานในส่วนนี้ไม่ได้มีแค่การส่งข้อความจาก backend ไปยัง LINE เท่านั้น แต่รวมถึงการออกแบบ flow การแจ้งเตือน การผูกบัญชีผู้ใช้กับ LINE การจัดเก็บ Group ID อัตโนมัติ การตั้งเวลาแจ้งเตือนด้วย cron job และการเชื่อมโยงกับข้อมูลวันหยุดและเวรทำงานวันเสาร์

### 1.2 วัตถุประสงค์ของการพัฒนา
1. เพิ่มช่องทางแจ้งเตือนที่ผู้ใช้งานเข้าถึงได้รวดเร็วผ่าน LINE
2. ให้ระบบส่งแจ้งเตือนอัตโนมัติจากเหตุการณ์สำคัญของงาน เช่น งานใหม่ งานใกล้ครบกำหนด และการเปลี่ยนสถานะ
3. เพิ่มการแจ้งเตือนเชิงปฏิบัติการภายในองค์กร ได้แก่ วันหยุดประจำปีและเวรทำงานวันเสาร์
4. ทำให้ผู้ดูแลระบบสามารถใช้งาน LINE group ได้จริง โดยระบบบันทึก Group ID อัตโนมัติจาก webhook
5. จัดทำระบบให้มีหลักฐานอ้างอิงชัดเจน สามารถอธิบายต่อในรายงานสหกิจได้

### 1.3 ขอบเขตของงาน
- การเพิ่ม `line_user_id` เพื่อผูกผู้ใช้กับบัญชี LINE
- การสร้าง service สำหรับส่งข้อความผ่าน LINE Messaging API
- การเชื่อม LINE notification กับ workflow ของงาน
- การพัฒนา webhook สำหรับบันทึก LINE group ID อัตโนมัติ
- การเพิ่ม cron jobs สำหรับแจ้งเตือนวันหยุดและเวรวันเสาร์
- การทดสอบและแก้ไขปัญหาเรื่อง timezone, broadcast และ error isolation

## บทที่ 3 สรุปผลการปฏิบัติงาน

### 3.1 สรุปภาพรวมการพัฒนา
การพัฒนาระบบแจ้งเตือน LINE ในโปรเจกต์ Intersite Track ไม่ได้เป็นเพียงการส่งข้อความทั่วไป แต่เป็นการต่อยอดระบบจัดการงานให้สามารถสื่อสารกับผู้ใช้งานได้ทันทีตามบริบทจริงขององค์กร โดยเริ่มจากการเชื่อม user กับ LINE, สร้าง service กลาง, ผูกเข้ากับ workflow ของงาน, เพิ่ม webhook สำหรับ group, และขยายไปสู่การแจ้งเตือนวันหยุดและเวรวันเสาร์ด้วย cron jobs

### 3.2 ผลลัพธ์ที่ได้จากการปฏิบัติงาน
1. ระบบสามารถส่งแจ้งเตือน LINE รายบุคคลเมื่อมีงานใหม่ งานแก้ไข งานเปลี่ยนสถานะ และงานใกล้ครบกำหนด
2. ระบบรองรับการแจ้งเตือนวันหยุดทั้งแบบวันปัจจุบัน วันถัดไป และสรุปรายสัปดาห์
3. ระบบรองรับการแจ้งเตือนเวรวันเสาร์ทั้งแบบส่วนตัวและแบบส่งเข้า LINE group
4. ระบบบันทึก Group ID อัตโนมัติผ่าน webhook ทำให้ deployment และ setup ง่ายขึ้น
5. ระบบมี script และเอกสารช่วยตรวจสอบและส่งมอบงานได้สะดวกขึ้น

### 3.3 ประโยชน์ที่ได้รับ
1. ได้เรียนรู้การเชื่อม external service ผ่าน LINE Messaging API
2. ได้ฝึกออกแบบ notification workflow ให้สอดคล้องกับการทำงานจริง
3. ได้ฝึกพัฒนา cron jobs และการจัดการ timezone สำหรับงาน production
4. ได้เรียนรู้การออกแบบระบบให้มีความทนทาน เช่น retry, error isolation และ dynamic configuration
5. ได้ฝึกจัดทำเอกสารที่เชื่อมโยงระหว่างโค้ดจริง หลักฐาน และผลลัพธ์การใช้งาน

### 3.4 ปัญหา อุปสรรค และแนวทางแก้ไข
**ปัญหา 1:** LINE bot เข้ากลุ่มไม่ได้หรือไม่มี Group ID สำหรับส่งข้อความ  
**แนวทางแก้ไข:** เปิดการอนุญาตให้ bot join group และเพิ่ม webhook เพื่อบันทึก Group ID อัตโนมัติ

**ปัญหา 2:** การส่งข้อความแบบ group มี regression และบาง event ทำให้ webhook ล้ม  
**แนวทางแก้ไข:** ปรับ LINE service และแยก error handling ใน webhook controller

**ปัญหา 3:** การคำนวณวันสำหรับ cron เสี่ยงผิดจาก timezone ของ server  
**แนวทางแก้ไข:** ปรับ helper `bangkokToday` และ schedule ทั้งหมดให้ยึด `Asia/Bangkok`

### 3.5 ข้อเสนอแนะสำหรับการพัฒนาต่อ
1. เพิ่มหน้าจอสำหรับผูก LINE account ของผู้ใช้จากในระบบโดยตรง
2. เพิ่มระบบเลือกเปิด/ปิดประเภทแจ้งเตือนรายบุคคล
3. บันทึกประวัติการส่ง LINE notification เพื่อใช้ตรวจสอบย้อนหลัง
4. เพิ่ม automated test สำหรับ cron logic และ payload ที่ส่งไปยัง LINE API
